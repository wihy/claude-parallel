## 架构全景

```
src/
├── __init__.py           (仅 __version__)
├── app/                  CLI 入口 (argparse + dispatch)
├── application/          编排与工作流 (orchestration/worker/merge/review/validation/context_extraction)
├── domain/               领域模型 (tasks / worker_result)
├── infrastructure/       外部设施 (claude/ input/ storage/ monitoring/ dashboard/)
└── perf/                 perf 子系统 (6 内部层)
    ├── protocol/         与设备通信 (reconnect/dvt/device)
    ├── capture/          原始数据采集 (sampling/webcontent/live_*/starters)
    ├── decode/           Schema 解析 (templates/timeprofiler/deep_export/time_sync)
    ├── locate/         ★ 符号定位层 (linkmap/atos/cache/resolver/dsym)
    ├── analyze/          高层分析 (power_attribution/ai_diagnosis/callstack/metrics/syslog_stats)
    └── present/          呈现 (report_html/dvt_metrics)
```

依赖方向: `app → application → domain ← infrastructure`; perf 只依赖 domain + infrastructure。
已通过 grep 静态检查验证所有 5 条分层规则无违反。

## 实时方法定位 (Track B 核心)

`SymbolResolver` 四层统一入口:
```
addr ─► cache (0.01ms) ─► LinkMap bisect (0.1ms) ─► atos daemon (2ms) ─► hex 兜底 (500ms timeout)
```

- `AtosDaemon` 常驻 `atos -i` 子进程，消除 500ms subprocess 启动开销
- `SymbolCache` LRU + 原子 JSON 落盘，session 间恢复
- `aggregate_top_n(filter_overhead=True)` 剔除 `dyld3::findClosestSymbol` 等 xctrace 采样自身开销（实测饱和 Top-10 达 45%）
- in-cycle 符号化，JSONL 每条带 `source` 字段 (cache/linkmap/atos/unresolved)
